How it works

One command to set up. Then it runs itself.

Switchman sits at the git level — not inside any specific editor. That's why it works with all of them today, without any plugins.

step 01

Init your repo

Run switchman init once. It finds your branches and sets up a shared task queue that all your agents read from.

step 02

Agents pick up work

Each agent calls task next to get its assignment. Tasks are handed out one at a time — no two agents ever get the same job.

step 03

Lock files before editing

Before touching a file, agents call claim. If someone else has it, they're told straight away and pick a different file instead.

step 04

Done — next agent's turn

When a task finishes, task done marks it complete and frees up the locked files so the next agent can get started.

Features

Everything you need. Nothing in the way.

No server to run. No Docker. No daemon. A single file in your repo and a CLI you already know how to use.

No double-assignments

Every task goes to exactly one agent. Even with a dozen running at once, no two agents ever pick up the same work.

File locking that works

Agents lock files before they edit them. If a file is already taken, the request is blocked immediately — not after the damage is done.

See everything at once

One command shows every agent, every active task, and every locked file — updated live so you always know what's happening.

Check before you merge

Run a scan before merging any branches. Switchman tells you exactly what would conflict so you can sort it out first.

Each agent on its own branch

Every agent gets an isolated branch to work in. Changes stay separate until you're ready to review and merge.

Nothing extra to install

Uses Node's built-in database. No packages to compile, no native binaries, no surprises on Mac, Linux, or Windows.

Claude Code

Your agents coordinate without any extra prompting

Switchman ships with a native Claude Code integration. Add two lines to your config and your agents will automatically pick up tasks, check for file conflicts, and release their locks when they finish — no coordination logic needed on your end.

It runs quietly in the background. Your agents just get on with the actual work.

task_next — picks up the next available task
task_claim — locks a file before editing it
task_done — marks complete, releases all locks
scan — checks for conflicts before merging
status — shows what every agent is doing right now
~/.claude/claude_desktop_config.json
{ "mcpServers": { "switchman": { "command": "switchman-mcp", "args": [] } } }

Add this, restart Claude Code, and your agents are coordinated.
