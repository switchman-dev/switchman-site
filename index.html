Why developers use it

More agents are easy. Trusting their output is hard.

Switchman gives you a legible record of what your agents built and a defensible answer to whether it's safe to ship.

⚠

The real risk isn't merge conflicts — it's agentic drift

Parallel agents can independently solve the same problem in incompatible ways. No conflict. No CI failure. Just divergence that compiles, passes, and breaks things later. Switchman detects semantic mismatches before they reach main — the thing git diff can't tell you.

The core feature

switchman review

After every session: a plain-English summary of what each agent built, semantic overlap flags, interface mismatches, and a green / amber / red merge confidence score. Run it in 30 seconds. Know what you have.

Honesty

Four confidence states

Green, amber, red — and uncertain. When Switchman can't determine coherence it says so. A tool that admits uncertainty is one you can trust when it says green.

Coordination

Invisible by design

File claims, lease heartbeats, stale recovery — all handled by the MCP server. Agents coordinate automatically. You never type a coordination command.

Visibility

One live dashboard

switchman status --watch — what's running, blocked, or done across all agents. The thing developers leave open and screenshot.

Cross-tool

Landing

One governed path to main

CI gate, merge queue, retries. Once review is green, work lands cleanly. No last-minute surprises at merge time.
